[译]我为什么弃 Mac 投 Windows：放弃 Apple

Posted on 2017-03-24，9 min read

time：2017-03-24

翻译：wolf3c

翻译字数：2.6K

译自：Why I left Mac for Windows: Apple has given up（点击查看原文）

我自己对 Windows 10 的满意度超过 Mac OS，一直想写点东西说明一下自己的观点（或者告诉那些无脑粉 Mac 的人一点不一样的东西），但是一直没有着手做。在看到这个标题后，我虎躯一震，决定翻译一下。

如果询问任何一个了解我的人，我大概是他们中最大的 Mac 死忠，如果有人问我购机建议，我一定会说 MacBook Pro，或者等下一代的 MacBook。

但是，最近我发现自己有点厌倦 Apple 对电脑桌面的态度，自从 2 年前发布了 Yosemite， Apple 对于 Mac OS 的更新聊甚于无，作为一个开发人员，在我看来， Apple 并不重视它的软件，更不要说投入 Mac 怀抱的成百上千开发人员。

看看 Sierra，更新日志中唯一值得注意的东西就是 Siri，并且是一个半成品，而且是从 iOS 中生搬硬套过来的那个半成品。对于开发人员？一无所有，除非你用 XCode——老生常谈。

Mac 作为 web 开发人员够用的平台的唯一原因就是，开源社区在 Mac 工具流上做的不懈努力。（想想 Node、npm、Yarn和其他新出现的工具多么易用）

彭博社曾 2016 年报道称 Apple 拆分了 Mac 团队，部分人转入了 iOS 团队，现在来看确实如此，2016 年底发布的新 MacBook Pro，虽然有点意思，但在用户方面的东西有点半吊子，它只能算是一个不错的东西，但并非非常棒，它的硬件性能不足，相比于 Mac 以前的特色——性能和功能，这次更关注轻薄和噱头十足的 Touch bar。

「另一方面， Apple 把他们的 Mac OS 团队重组为软件工程团队，以示对 iPhone 的重视。 」

实话实话，在让人失望的 Windows Vista 前，我是一个痴迷于 Microsoft 的 Windows 死忠，很明显，即使 Windows 7 掩盖了一些问题，但它依旧是一个没有战略和愿景的公司。Apple 却拥有我想要的愿景——一切工具流畅协作。

你可以在电脑或手机上发送 iMessage，随时随地回复来电，方便地把文件放到其他设备中。总而言之，我被 Apple 在 2013 年初发布的 15" Retina MacBook Pro 深深吸引。

我在这台电脑初探 Apple 世界，并为之着迷。一切设计恰到好处，一切都协作流畅。Apple 基于 Unix，所以我拿起来就可以做 web 开发，我知道如何深入使用终端，或者马上进入那些开源项目。

但是，当我兴冲冲地加入 Apple 世界，这家公司的重心发生了变化。Mac 不再重要，同时 iPhone、iPad、以及后来的 iPad Pro 成为了 Apple 的重心，信号很明显：为什么你非要用电脑来做一个手机或平板就可以搞定的事情呢？

结果就是 Apple 不在关注 Mac，硬件四年都没有得到一点儿更新。还有 OS X，以前还能是不是更新一些有趣的新功能，现在只能拿到一些 iOS 的边角料，如果你不信，看一下你 Mac 中的 iMessage，iOS 的旗舰功能——贴纸在 Mac 中毫无踪迹。

这点事无处不在，比如 Airdrop——Apple 非常受欢迎的功能，轻松在电脑和手机间传递文件的体验让我感受到了什么叫「浸入式工作」，要知道，每次传递文件耗费了我 30% 的时间，而其他时间我都在连接设备。有了 Airdrop 后，你就可以在一个设备上工作，无缝地移动到另一个设备上。很想知道用这个功能是怎样的感觉。

在此期间，Microsoft 舔好了 Windows 8 的伤口，找到了一个激情昂扬的 CEO——Satya Nadella，然后在 Windows 10 中做了很多非常棒的东西——它开始倾听并实现用户需要的功能了。

我一直无视 Windows 世界，直到去年底，Microsoft 在 Windows 中推出了 Linux 子系统（简单说，就是一种可以在 Windows 中使用 Linux 终端的方式），这让我意识到在 Windows 开发将变得更加友好，这足以让我回心转意。它是一个游戏的改革者：

在它（译注：Windows 10）的内核中，在 Windows 上支持 Bash 既激动人心，又是一个难以置信的工程壮举，作为一个前端开发人员，我现在考虑买一台 Windows 电脑代替我的 Macbook，这在以前不可想想。

经过对 MacBook 更新的漫长等待，等来的是对于 Apple 所交答卷的无限失望，花大价钱买了个低配货，我开始想要回到 Windows了。在 Mac 世界，美好生活逐渐消失的迹象从 2015 年就已显现，整整一年，Apple 也没有对硬件方面带来一点有意义的冲击。尤其是当时我已对虚拟现实兴趣日浓，但 Oculus 因为 Apple 的表现而放弃对 Mac 的支持。

到了 2016 年，Microsoft 推出新版 Windows「创新永无止境」( 原文：Creators Update out of nowhere )，包括专用游戏功能、完整系统级 VR 支持、颜色自定义、语音助手以及其他一大堆更新。

我被惊得合不拢嘴（这么多年来，Microsoft 第一次让我不愿错过它的任何动态），我很好奇 Microsoft 是如何在短时间内做出如此令人惊叹的功能。

就是这样，在看到 Microsoft 正在做的 Linux 子系统，以及看到对面苹果那些毫无意义的进步，这让我决定做一次大反水。我并不是讨厌什么，我只是不希望自己使用是一款不值得被我使用的设备。

那些站出来抱怨 Apple 现有 Mac 的人不但不是 Mac 黑，反而是 Mac 粉，他们已经在 Mac 上花费了十几年时间和数十万元

我用好几个月才说服自己（译注：从 Mac 到 Windows），但在我花几周时间逛了论坛中关于电脑参数和新硬件的帖子后，我就意识到 PC 超前多远：NVIDIA GTX 1080 显卡就像一个性能怪兽，不论是什么游戏或是 VR 之类的，它都可以轻松搞定。

我意识到自己对 Apple在笔记本和台式机方面的平庸 已经厌烦透顶，然后开始考虑再次尝试使用 Windows。

所以，2017 年 2 月，我开始从零组装一台电脑，我卖掉了自己的 15" MacBook Pro（我现在用一个 12" MacBook 移动办公），并持续投入精力建立一个适合自己需求的桌面新主角。

我将省去很多细节，如果你对我的作品感兴趣，可以查看我订购的配件，设备有点变态，但是考虑到我想学习虚拟现实开发，从长远来看这似乎是性能和价格之间最好的平衡。

（Jekyll 和 Gulp 和谐相处）

如今，我再 Windows 中工作了差不多 6 周了，虽然我希望自己可以讨厌它，但我发现我记忆犹新，虽然它不完美，但它清晰地展示出 Microsoft 绝无仅有的细节，我是按照我再 Mac 中的习惯建立开发环境的，从 Jekyll 到 Gulp 一切都如我所愿，太感谢 Linux 子系统了。

我一开始比较担心 Microsoft 对高分辨率的支持，毕竟在 Windows 8 和 Windows 10 原有版本中都没有实现，但在新版（Creators' Update）中貌似已经可以毫无压力支持 4K 显示器了。

还有更重要的？那就是我可以任意玩各种游戏，我可以体验 VR，HTC Vive 是一台不可思议的设备，我已经开始学习 Unity 去创建自己的应用。

并非所有的经历都是如此浪漫，我忘记了硬件是一回事，而 Windows 的软件非常匮乏（虽然比我记忆中好了很多），这一点很快会被 Electron apps 解决，比如 Slack、Nylas、 Hyper 以及 Visual Studio Code 都是非常优秀的，他们将在 Windows 世界脱颖而出。

（看似一小步，其实一大步）

虽然我不愿明说，但 Windows 真的回来了，Microsoft 干的漂亮。Microsoft 正在让 Windows 比 Apple 的 OS X 在各个方面更快、更好。

还有一些我忘说的事情：Sketch，我经常用它做设计；iMessage，我可以不用不停地拿出手机；……不过离开他们并不难。

我对 Apple 从内而外看的一清二楚，整个 Apple 都只关心 iOS 和 iPhone（咳咳，忘了 Airport），或许有一天他们会不知道从哪里拿出一个令人惊讶的产品超常发挥，但是这个公司正忙于说服用户他们的重磅产品 iPad Pro 是有利于工作的。

我计划在未来几周写一些关于 Windows 的文章，包括我如何配置自己的开发环境、我使用的软件，当然也包括不好的方面。老实说，我并没法在工作中放弃 Mac，因为我不确定有一款很好的 Windows 笔记本电脑…但这可能随时改变。

如果你是一个坐下来等 Apple 可能发布一款让你工作的主力电脑的人，放弃无谓的挣扎吧，看看 Windows 多么棒，并且，现在你就可以用超实惠的价格拿到一个超高性能的电脑去使用。

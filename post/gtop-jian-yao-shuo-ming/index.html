gtop 简要说明

2019-09-09 app ， 技术

在 *nix 系统中，top 命令的用途不言而喻，如果有人不了解，可以将他类比 Windows 的任务管理器的部分功能，主要用于查看系统资源使用情况，是进程管理中非常重要的工具。

不过，所有被大量用户使用的工具，总是不能很好地满足小部分用户的小众需求，top 也是一样，一些人希望 top 的能力更强大，所以有了 htop，也有一部分人，希望 top 的功能尽量简单易上手，所以就有了 gtop。

可能大家都希望拥有更强大的工具，所以网上有很多介绍 htop 的文章，但很少有人介绍 gtop，可能是他简单到无需介绍吧。不过作为一个简单看看电脑资源的人，我还是挺喜欢 gtop 的。

首先 gtop 很直观地列出了重要资源信息：

平时重点关注的内存、硬盘、网络情况，都一目了然。

另外，平时想要针对性地看看进程占用资源的情况（上图右下角区域的内容），gtop 提供了三个快捷键，覆盖了日常需求：

c： 将进程按照 CPU 使用率排序
m：将进程按照内存占用情况排序
p： 将进程按进程 ID 排序

所有的排序，第一次点快捷键是倒序，再点一次就是正序。电脑发烫，那看看哪个应用在疯狂用 CPU，进入 gtop 点一下c；电脑老是提示内存不足，那看看哪个应用吃内存，进入 gtop 点一下m。

没事时，也可以直接打开 gtop 放在那里，等着装逼，装完，按一下q或Esc就可以退出。

如果感兴趣，也可以自己去试试他，由于他是基于 node.js 的应用，Windows、Mac、Linux 都支持：

npm install gtop -g
